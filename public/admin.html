Nuevo factor de precio

¿Qué es un factor de precio?

Cada servicio tiene un rango de precio base (min–max). El factor define las preguntas que se hacen al cliente en el cotizador y cómo modifican ese precio.

Por ejemplo: si un servicio vale Bs 800–2500, el factor puede preguntar "¿Urbano o rural?" y el multiplicador ajusta el precio final.

Debe coincidir exactamente con el campo details_type del servicio.
Ejemplo: si el servicio tiene details_type = topo_area, el ID del factor debe ser topo_area.
Nombre que se muestra internamente.

Estructura JSON de Parámetros

Cada parámetro es un objeto dentro del array JSON. Tipos disponibles:

  • "tipo": "chips" — Selección única. El cliente elige UNA opción y aplica su multiplicador.
  • "tipo": "chips_multi" — Selección múltiple. El cliente puede elegir varias opciones y cada una suma un precio extra.
  • "tipo": "number" — Campo numérico. Útil para cantidad de puntos, hectáreas, etc.

Multiplicadores (para tipo chips):

  • 1.0 = precio base (sin cambio)
  • 0.85 = 15% más barato (proyecto sencillo)
  • 1.3 = 30% más caro (proyecto más complejo)
  • 1.8 = 80% más caro (proyecto muy complejo)

Precio extra (para tipo chips_multi):

  • "precio_extra": 500 = suma Bs 500 al total por cada opción seleccionada
  • "multiplicador": 1.0 = no modifica el multiplicador base
